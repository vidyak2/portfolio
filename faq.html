FAQ

What is the process between typing a URL into a browser and the site finishing loading?

On a day-to-day basis, website names are typed into the browser of a device, such as a computer, a tablet, or a phone and the user expects the browser to return the contents of the website. Each website is identified by a Uniform Resource Locator (URL), which is directly related to a resource on the web. A single URL is usually linked to a single resource (MDN web docs, 2018). Between the moment when a URL is typed, and the site loads, many steps have to happen on the back-end. Though this process seems simple, it consists of a collaboration of many elements.

The process between typing the URL and the website loading is a relationship between the client and the server, where the client is any device that is connected to the internet and the server is the base where websites and apps are contained. It is important to note that having an internet connection is necessary, since it permits the sending and receiving of data. The client and the server interact in a way that the client sends requests to the server, and the server answers to the needs of the client (MDN web docs, 2017). Although this is the crux of the process, many elements contribute to accomplish the task of loading a webpage on the device of the user.

The process between typing the URL and the loaded website appearing can be explained in five steps (MDN web docs, 2017). The first step is typing the web page name into the browser. The web page name consists of the protocol and the domain name. For example, the URL http://www.google.com can be separated into the protocol, which is http, and the rest is the domain name. The protocol indicates the method of communication within the computer network, and is usually “http”, and “https” in a secure browser. The HTTP protocol will be described further below. In this example, the domain name is “www.google.com”, which is the unique name of the website (MDN web docs, 2018).

The second step is finding the real address of the server where the website is present. This is done with the help of the domain name server (DNS), which links the web address typed as a string in the browser, to the correct IP address. The IP address is the location where the page can be found, and this is represented by a compilation of numbers. This address follows the format: #.#.#.#.

The third step is sending an HTTP (Hypertext Transfer Protocol) request message from the browser to the server. This will allow a version of the website to be available to the client. The HTTP sets the language of communication between the client and the server. These messages are sent through internet connection, using the Transmission Control Protocol (TCP) and the Internet Protocol (IP).

The fourth step is sending the website files from the server to the browser. This is sent in small pieces, called data packets.

The final step is that the browser will put all the pieces in order, and present the website (MDN web docs, 2017).
